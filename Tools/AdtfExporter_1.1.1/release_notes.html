Release Notes for ADTF Exporter

1.1.1 2018-03-02

Changes:

  • MKS 671833: Add subsecond support in offset option
  • MKS 647525: Add possibility to parse Automotive Ethernet with new major type

1.1.0, 2017-11-17

Changes:

  • MKS 493812: Add support for Automotive Ethernet
  • MKS 645447: Add time offset option for ASC timestamps

1.0.0, 2014-12-11

Changes:

  • MKS 182304: Add support for extended CAN

0.9.1, 2013-10-14

Changes:

  • MKS 132217: Fixed bug which caused incorrect FlexRay timestamps in ASC file

0.9.0

Changes:

  • MKS 114033: Added support for output of FlexRay data in ASC format
  • Removed output of video chunk timestamps

0.8.0

Changes:

  • Added batch result logging.
  • Added --log-path flag to control where the batch result log is written.
  • Refactored the code a bit.

0.7.0

Changes:

  • Added --output-path flag.
  • Rewrote the tool to use the CommandLineApplication framework.
  • The tool is now compiled as a .NET 4.0 application.

0.6.2

Changes:

  • Added imagetimestamps for generic stereo streamnames.

0.6.1

Changes:

  • Fixed header and footer bug when merging CAN data and outputing ASC file format.

0.6.0

Changes:

  • Added support for outputing ASC file format using the --output-asc option.

0.5.0

Changes:

  • Added support for appending video time stamps using the -videotimestamps option.

0.4.0

Changes:

  • Added support for specifying filelist files are input arguments using the @ character as prefix to the filename.

0.3.0

Changes:

  • Added support for merging CAN channels to the same output file using the --merge-can option.

0.2.0

Changes:

  • Added support for wildcard and environment variables in file names
  • Added progress and timing output to console

0.1.0

Changes:

  • Initial release
